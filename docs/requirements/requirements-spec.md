# 需求规格定义

> 阶段三产出 | 基于用户确认的场景和优先级
> 更新日期：2026-06-18

## 用户编曲场景基线

**涉及风格：** 电子音乐、流行音乐
**典型工程规模：** 15-20 轨/首

| 轨道类型 | 数量 | 说明 |
|---------|------|------|
| 氛围 Pad | 2+ | 和声铺底 |
| 琶音/分解 Arp | ≤3 | 钢琴/电吉他/吉他/小提琴/拨弦 |
| 鼓组 Drums | 2+ | Kick / Snare |
| 小打/过渡 | 1 | 过渡音效 |
| 拨奏 Pluck | ≤3 | 主要律动元素 |
| 弦乐 Strings | ≤4 | 情感层次 |
| Bass | 2 | 低音 + 中音 |
| Melody 主旋律 | 2 | 核心旋律线 |

## 需求清单（MoSCoW）

### Must Have（MVP 必须）

#### M1: 口述/语音 → 鼓组写入
- **描述：** 用户口述鼓组节奏（如"底鼓在 1、3 拍，军鼓在 2、4 拍"），Agent 解析后写入 FL Studio Step Sequencer
- **验收标准：**
  - 支持中文口述描述
  - 能写入 Kick、Snare、Hi-hat 至少 3 种鼓件
  - 写入后可在 FL Studio Step Sequencer 中看到结果
  - 支持 4 小节以上的 pattern
- **输入：** 自然语言文本 / 语音转文本
- **输出：** FL Studio Step Sequencer 中的鼓点数据

#### M2: 口述/语音 → 旋律写入
- **描述：** 用户口述或哼唱旋律，Agent 解析后写入 FL Studio Piano Roll
- **验收标准：**
  - 支持中文描述（如"A 小调，do re mi sol la"或音名"A B C D E"）
  - 支持指定调性、拍号、小节数
  - 写入后可在 Piano Roll 中看到音符
  - 写入的内容可手动修改（不是冻结的）
- **输入：** 自然语言文本 / 语音转文本
- **输出：** FL Studio Piano Roll 中的音符数据

#### M3: MCP 基础通信
- **描述：** MCP Server 能通过 MIDI SysEx 与 FL Studio 建立双向通信
- **验收标准：**
  - MCP Server 启动后能连接虚拟 MIDI 端口
  - 能发送命令到 FL Studio 并收到响应
  - 错误时有明确的错误信息
  - 支持 Windows（loopMIDI）

#### M4: 中文自然语言理解
- **描述：** Agent 能理解中文音乐术语并转换为结构化指令
- **验收标准：**
  - 理解调性描述：A 小调、C 大调、降 B 调等
  - 理解拍号：4/4、3/4、6/8
  - 理解音符时值：全音符、半音符、四分音符、八分音符、十六分音符
  - 理解鼓组描述：底鼓/军鼓/踩镲/开镲
  - 理解和弦进行：1645、6415 等数字标记

---

### Should Have（应该有）

#### S1: Mixer 基础控制
- **描述：** 用户可以通过 Agent 调节 Mixer 音量、声像、静音/独奏
- **验收标准：**
  - 能设置指定轨道的音量（绝对值或相对增减）
  - 能设置声像（左/右/居中）
  - 能静音/独奏指定轨道
- **优先级：** MVP 之后第一个

#### S2: 工程整理
- **描述：** Agent 可以重命名 Pattern、改色、切换当前 Pattern
- **验收标准：**
  - 能重命名 Pattern（如"改名 Pad"）
  - 能改色（如"涂红色"）
  - 能切换到指定 Pattern

#### S3: 语音实时输入
- **描述：** 支持语音模态的实时灵感捕捉
- **验收标准：**
  - 能接收语音输入（哼唱或口述）
  - 语音转文本后走标准流程
  - 延迟可接受（< 5 秒从说话到写入）
- **依赖：** 语音识别服务（Whisper / mimo-omni 等）

---

### Could Have（可以有）

#### C1: 和弦进行生成
- **描述：** 用户指定调性和进行标记（如"A 小调 1645"），Agent 生成完整和弦并写入
- **验收标准：**
  - 支持数字标记法（1645、6415、4536 等）
  - 支持指定小节数和每个和弦时值
  - 写入 Piano Roll

#### C2: Bassline 生成
- **描述：** 根据和弦进行或鼓组自动生成 Bass 声部
- **验收标准：**
  - 能跟随和弦根音
  - 支持指定节奏型（八分音符、四分音符等）

#### C3: 琶音生成
- **描述：** 指定和弦进行和琶音模式，自动生成琶音
- **验收标准：**
  - 支持上行/下行/随机琶音模式
  - 支持指定音域和速度

---

### Won't Have（暂不做）

#### W1: Playlist clip 排列
- **原因：** FL Studio API 不支持，需探索 PyFLP 绕路

#### W2: 自动导出/渲染
- **原因：** FL Studio API 不支持

#### W3: 第三方插件参数控制
- **原因：** 插件 UI 无法脚本化，只能操作有参数暴露的插件

#### W4: AI 听觉反馈
- **原因：** 需要接入音频分析模型，复杂度高

---

## 非功能需求

| 维度 | 要求 |
|------|------|
| 延迟 | 从指令到写入 FL Studio < 3 秒（不含语音识别） |
| 兼容性 | Windows 10/11 + FL Studio 21+ + loopMIDI |
| 可用性 | 中文交互，无需编程知识 |
| 可维护性 | 模块化设计，各层可独立替换 |
| 扩展性 | 新增 MCP 工具不需要改通信层 |

---

## MVP 功能边界

```
MVP = M1(鼓组写入) + M2(旋律写入) + M3(MCP通信) + M4(中文理解)
```

MVP 的唯一验证目标：
> 用户用中文口述一段描述（如"A 小调，do re mi sol la，4 小节"），Agent 在 FL Studio 的 Piano Roll 里写入对应的音符。
